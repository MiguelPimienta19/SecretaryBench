# grader.py — Systems Grader

`grader.py` exposes a single function, `define_grading_system(input, calendar, todo)`, which takes either an `Email` or `Scenario` object along with a `CalendarResponse` (from `app.models.calendar`) and a `list[TodoResponse]` (from `app.models.todo`) and returns a score dict. It first checks whether the input is a single email or a full scenario — if it's an email, it wraps that email's `success_criteria` string into a list; if it's a scenario, it uses the already-collected `success_criteria` list directly. It then iterates through each criteria entry and awards 1 point for any non-empty string. The `CalendarResponse` and `TodoResponse` parameters are accepted but not yet inspected — the stub scoring logic is a placeholder that the team will replace with real checks against the Week 2 tool APIs (verifying that todos were created, calendar events were scheduled, etc.) once the full grading strategy is defined.
